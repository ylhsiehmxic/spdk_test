用 lifecycle 看就更清楚...
對 BDEV_IO：
BDEV_IO_START   ← new_object = 1
BDEV_IO_SPLIT   ← new_object = 0
BDEV_IO_SUBMIT  ← new_object = 0
BDEV_IO_DONE    ← new_object = 0


每一層：
都會 new 一個自己的 IO struct
都會 new 一個自己的 trace object
都會 new_object = 1
然後用：
spdk_trace_tpoint_register_relation()
把 parent object 和 child object 串起來。


spdk_trace_tpoint_register_relation()的第三個參數, 0或1, 什麼意思呢
原程式：
bdev:
	spdk_trace_tpoint_register_relation(TRACE_BDEV_NVME_IO_START, OBJECT_BDEV_IO, 0);
	spdk_trace_tpoint_register_relation(TRACE_BDEV_RAID_IO_START, OBJECT_BDEV_IO, 0);
raid:
	spdk_trace_tpoint_register_relation(TRACE_BDEV_IO_START, OBJECT_BDEV_RAID_IO, 1);
答：
例如spdk_trace_tpoint_register_relation(TRACE_BDEV_RAID_IO_START, OBJECT_BDEV_IO, 0);
前者這個 event 的第 0 個 argument 是後者的 object id
